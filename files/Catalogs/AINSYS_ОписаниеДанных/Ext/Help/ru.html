Содержит правило синхронизации данных со сторонней системой. В интерфейсе используется внутренний идентификатор элемента справочника AINSYS_ОписаниеДанных.

В табилце "Описание данных" описывается структура дерева значений в которое соберутся отдаваемые данные. Точно такое же дерево значений ожидается при загрузке.

Наименование элемента должно удовлетворять правилам именования переменных в 1С.

Описание сущности показывается  на странице сервиса. Это описание для специалиста корреспондирующей системы.

Флаг Отключено используется для прекращения использования правила.

Способ выборки данных может принимать значение Схема компоновки данных (СКД) или Программная. При заполнении исходящих данных с помощью СКД необходимо соблюдать следующие правила:

  • СКД использует только один вариант отчёта.
  • Структура отчёта должна содержать только одну корневую группировку. Подчинённых группировок можеи быть несколько на одном уровне. В каждой группировке должны быть явно перечислены все поля. Каждой группировке необходимо присвоить имя.

Таблицу Описание данных можно заполнить по СКД либо вручную. Эта таблица показывается на странице сервиса. По сути это документация к интерфейсу.

Если при записи данных придёт структура с неописанными полями, то они появятся справа от описания данных.

Обработка выгрузки может быть пустой, если данные получаются СКД. Выборку данных можно формировать полностью программно. Или сочетать. Сначала данные выбираются СКД, затем дерево значений ВыборкаВыгрузки обрабатывается программно.

Обработка загрузки выполняется только программно. Входящие данные приводятся к формату в дереве Описание данных. Поля с типом Ссылка преобразуются в ссылки внутри базы данных. 

Обработка отправки данных вызывается при инициации обмена.
